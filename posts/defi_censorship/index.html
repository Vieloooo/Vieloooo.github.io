当用户相信 Defi 与 传统金融服务具有相同的安全性时会发生什么?

Created on Oct 17, 2023

本文探讨的 defi 均是指基于账户模型智能合约形式 defi protocol: uniswap, aave, compound, maker dai 等.

  • 主流 defi 应用的合规化是不可避免的
    • 当今主要的 defi 合约都部署在基于 PoS 相关算法的 L1 或是基于此类 L1 的 rollup 上
      • L1 质押中心化
      • L1 结点的中心化
    • Defi 治理团体的实体化
      • 例子:
        • Makerdao RWA
        • DAO as LLC at some states in the US
    • defi 本质是提供一种金融服务, 所有服务必然 Liable, 即服务的提供方必须对自己提供的服务负有责任.
      • 在一个法治国家, 现实生活中所有的服务都是需要接受监管, 即使该服务是非盈利性质的.
      • 监管是对"民众对服务安全性的要求"的共识的执行.
    • 前两点阐释了合规化的可实现原因, 最后一点实现了合规实现的动机
  • 合规化/审查可能会发生在哪些层面?
    • Consensus Layer:
      • 对 validator 的审核(比如 kyc);
      • 对 p2p 网络的审核 (屏蔽某些地区的结点);
      • 所有结点对交易进行一定审核(相当与一套对交易的共识规则);
      • 配合监管方进行 fork, 类似 2016 The DAO Fork;
    • Execution Layer: 单个结点( sequenser, block builder…) 对交易的审核
    • Protocol Layer: defi protocol 对交易进行审查
    • Frontend: UI 前端对交易进行审查
  • 那些实体有被监管的风险?
    • 所有使用者
    • Governance Token 持有者
    • 受雇佣于某个 DAO 的实体: 开发者, 律师, 交易员等等.
  • 如果要对 defi 达到传统金融服务相同的监管力度, 需要哪些层面的监管?
    • 不幸的是, 如果我们仅仅思考现实世界 “银行经营需要政府颁发执业许可” 这一条要求, 就会发现我们需要在 consensus layer 实现审查. 具体来说, 制定一条如下的规则:
      • “审查每个上链的合约, 判断是否为银行合约(lending protocol like aave, compound…), 如果是的话就要检查该合约是否合规”
      • 而"合规性检查"进一步会对该 protocol 的发行方进行审查(显然需要发行方进行 KYC)并要求该 protocol 满足反洗钱要求.
  • 悲观的看, 如果政府使用相同的监管力度约束 defi 服务, 那么 defi 寄生的链将会直接变成一条联盟链, 所有 defi 的用户都要进行 KYC.
  • 幸运的是, 区块链在执行层上的无需信任性与透明性(或是可证明性在某些隐私链上)给予了 defi 更为宽松的监管力度, 使得 defi 可以在更低的审查力度下, 更低的监管成本下运行.
    • 让我们再回顾一下政府(民选)监管的目的: 让民众感到安全. 这是一个非常主观的而且变化的指标.
    • 面对一个每时每刻公开自己的账本的金融服务商(而且保证为真)和一个每年公开自己账本(还不知道增加)一次的服务商, 前者显然在更宽松的监管下可以给与后者一样安全感.

总而言之, 相较于传统金融服务, defi protocol 依靠其无需信任性与透明性溢价, 换取了更为宽松的监管力度, 从而进一步换取在流动性与开放性的优势. 至于能换来多少优势, 取决于到底要大多的监管力度才能让用户感到安全. 下面是几种可能的形态:

  • defi running on decentralized, secure but extremely slow blockchain => no censorship => 开放性 +++, 流动性–
  • defi running on perminssionless PoS chain, frontend censorship => 开放性 +++, 流动性+.
  • defi running on perminssionless PoS chain, protocol level blacklist/kyc, defi governance DAO fully liable => 开放性-, 流动性+
  • defi running on perminssioned chain, consensus level censorship => 开放性–, 流动性随着准入门槛的提高而下降
  • tradfi + permissioned chian, fully off-chain kyc => 开放性—, 流动性-=> only make SEC’s live easier
  • defi
